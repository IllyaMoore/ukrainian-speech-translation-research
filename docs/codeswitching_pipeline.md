# Entity-aware code-switching pipeline для зменшення каскадного ефекту

## Постановка задачі

Каскадний pipeline `Whisper -> OPUS-MT` на корпусі українського
політичного дискурсу дає BLEU = 19. Аналіз помилок показує, що 46% усіх
спотворень припадає на власні назви, ще 16% --- на географічні. Кореляція
між CER (помилками ASR) та BLEU становить r = -0,59 (p = 0,042), що
підтверджує наявність каскадного ефекту: помилки розпізнавання
поширюються на етап перекладу.

Гіпотеза: якщо виправити власні назви до подачі в OPUS-MT, якість
перекладу має зрости.

## Метод

Pipeline працює у чотири кроки:

1. **Виявлення сутностей у UA-тексті ASR**: двопрохідний пошук.
   Перший прохід шукає точні підстроки у словнику UK->EN.
   Другий прохід для незахоплених позицій застосовує фонетичну дистанцію
   (транслітерація + Levenshtein, поріг 0,18) для пошуку спотворених
   варіантів власних назв, що могли утворитися внаслідок ASR-помилок.

2. **Code-switching**: знайдені сутності замінюються в UA-тексті
   на канонічну англійську форму прямо в позиції джерела.
   Приклад: `Рустему Міров на зустрічі`  ->  `Rustem Umerov на зустрічі`.

3. **Переклад через OPUS-MT**: модель MarianNMT, навчена на uk-en
   парах із Latin proper nouns, переважно копіює такі токени.
   У сирому вигляді 62% форм виживає; решта спотворюється моделлю
   (Rust Umerov замість Rustem Umerov, Sumland замість Sumy region тощо).

4. **Post-MT repair**: для кожної EN-форми з вставлених шукаємо у виході
   ngram-вікно (1-3 слова) із мінімальною Levenshtein-дистанцією
   (поріг 0,30 на латиниці). Якщо знайдено матч, спотворене вікно
   замінюється на канонічну форму. Survival rate зростає з 62% до 75%.

## Чому не плейсхолдери

Перша версія використовувала UPPERCASE-маркери (XQQAA, XQQAB, ...).
Гіпотеза полягала в тому, що рідкісні токени проходять через MT
без змін. На практиці BPE-токенізатор OPUS-MT розбиває їх на subwords;
після генерації лише 30% плейсхолдерів виживало в перекладі (виходило
`XQABA`, `XQAJ-SKybblebicK`, `XQQQAH`).

Code-switching розв'язує цю проблему: латинські proper nouns є
частиною тренувального розподілу uk-en моделі, тому копіюються краще.
Survival rate зростає з 30% (плейсхолдери) до 62% (code-switching) і
далі до 75% із post-MT repair.

## Технічні деталі

### Транслітераційна метрика

Для фонетичного матчингу використовується спрощена BGN/PCGN
транслітерація з нормалізацією поширених ASR-плутань:

- `и/і/й/ї` -> `i` (всі i-подібні звуки)
- `г/х` -> `h` (h-подібні)
- `ь/'` -> nothing
- щ -> ш, ё -> е

Дистанція обчислюється як нормований Levenshtein на отриманих
транслітераціях. Поріг 0,18 калібрується на 3 валідаційних сегментах
з мінімізацією WER.

### Доменний словник

176 записів у 5 категоріях, побудованих ручним аналізом референсних
перекладів 12 сегментів корпусу:

- Персоналії (62): Рустем Умєров -> Rustem Umerov, Кирило Буданов -> Kyrylo Budanov
- Географічні назви (55): Кривий Ріг -> Kryvyi Rih, Дніпровщина -> Dnipropetrovsk region
- Країни та міжнародні організації (30): Євросоюз -> European Union, Сполучених Штатів Америки -> United States of America
- Установи та абревіатури (15): СБУ -> SBU, ГУР -> HUR, ДСНС -> State Emergency Service
- Військові підрозділи та інше (14): окремого штурмового батальйону -> Separate Assault Battalion

### Post-MT repair

Алгоритм fuzzy-substitution на латиниці. Для кожної вставленої EN-форми:

- Якщо вона є в перекладі дослівно --- пропускаємо.
- Інакше перебираємо ngram-вікна довжини N-1, N, N+1 (де N = кількість
  слів у EN-формі).
- Для кожного вікна обчислюємо нормовану Levenshtein-дистанцію.
- Якщо знайдено матч з дистанцією < 0,30 --- замінюємо вікно на
  канонічну форму.

Поріг 0,30 обраний як компроміс між recall (більше 0,3 виправляє "Rust
Umerov" і "Sumland") та precision (більше 0,4 починає false positives).

### Захист від хибних спрацювань на етапі виявлення

Стоп-список загальних слів (україни, україна, росія, америка тощо)
для phonetic-проходу, щоб не сплутати "українці" з "Україна".

Skip exact-match canonical: якщо слово точно збігається з канонічною
формою (правильна словоформа), не чіпається.

Greedy довшим: перебираються вікна 1->3 слів, обирається найдовше
співпадіння (пріоритет багатослівних entity).

## Результати

Інтервенційний експеримент на 12 сегментах корпусу:

| Метод                                     | BLEU  | BERT-F1 | METEOR | Entity F1 |
|-------------------------------------------|-------|---------|--------|-----------|
| OPUS-MT (baseline ASR)                    | 19,01 | 0,330   | 0,416  | 0,447     |
| OPUS-MT (exact-match)                     | 19,13 | 0,344   | 0,421  | 0,455     |
| OPUS-MT (phonetic A)                      | 18,93 | 0,330   | 0,415  | 0,450     |
| OPUS-MT (LLM-corrected)                   | 19,51 | 0,340   | 0,429  | 0,451     |
| OPUS-MT (code-switching, 52-dict)         | 19,62 | 0,346   | 0,418  | 0,515     |
| **OPUS-MT (code-switching, 176-dict)**    | **20,33** | **0,350** | 0,423 | **0,610** |
| Claude Haiku 4.5 (direct)                 | 55,25 | 0,724   | 0,722  | 0,837     |
| Claude Sonnet 4 (direct)                  | 55,46 | 0,710   | 0,730  | 0,849     |
| Gemini 2.0 Flash (direct)                 | 56,92 | 0,739   | 0,747  | 0,845     |
| GPT-4o (direct)                           | 57,00 | 0,725   | 0,746  | 0,860     |

Code-switching з розширеним словником (176 записів) дає найбільший приріст
серед усіх локальних методів post-ASR корекції:

- Entity F1 зростає з 0,447 до 0,610 (+36% vs baseline)
- BLEU зростає з 19,01 до 20,33 (+6,9%)
- Wilcoxon signed-rank vs baseline: p = 0,046 (статистично значуще;
  єдиний серед локальних методів)
- Стандартне відхилення BLEU знижується з 4,34 до 2,30 (вдвічі стабільніше)
- Виявлено 160 сутностей у 12 сегментах; survival rate post-repair 75%

Метод закриває 40% розриву між baseline OPUS-MT та GPT-4o за Entity F1.

У середньому всі чотири протестовані LLM (Claude Haiku 4.5, Claude
Sonnet 4, Gemini 2.0 Flash, GPT-4o) перевершують запропонований метод за
Entity F1 (0,837--0,860 проти 0,610). Однак посегментний аналіз показує,
що на 2 з 12 сегментів (17%) метод перевершує Claude Haiku 4.5 за
Entity F1 (Діалог з Америкою seg001: 0,828 vs 0,800; Наша стратегія
seg001: 0,818 vs 0,723); ще на 2 сегментах (17%) досягається рівність.
Отже, на 33% сегментів корпусу локальний метод конкурує з LLM найменшого
класу.

## Обмеження

1. Метод залежить від словника. Сутності, яких немає в dict, не
   виявляються (на відміну від справжнього NER-підходу). Розширення
   словника дає лінійний приріст покриття.

2. Survival rate 75%: репер не виправляє радикальні спотворення
   (Levenshtein > 0,30) або випадки, де токен повністю зникає з виходу
   OPUS-MT.

3. У середньому метод не досягає рівня LLM. Абсолютний розрив з
   найслабшою LLM (Claude Haiku 4.5): 0,837 vs 0,610 за Entity F1.
   Це відображає фундаментальну перевагу моделей з ширшими
   контекстуальними знаннями про світ.

## Коли застосовувати

Підходить для real-time / low-latency сценаріїв (1 с/сегмент),
off-line обробки корпусів без доступу до API, задач із критичною
точністю передачі іменованих сутностей (новини, дипломатичні тексти),
доменів з обмеженим набором сутностей.

Не підходить для загального тексту без іменованих сутностей,
доменів з великою кількістю невідомих entities, випадків, де якість
важливіша за вартість і латентність (краще LLM-MT).
